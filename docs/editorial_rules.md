# Clearforge Editorial Rules

## Core rule

Clearforge does not publish raw AI output.

Every brief, article, and social post is a draft until a human reviewer approves it.

## Required checks before publishing

Check:

- source links work
- source dates are current
- facts match the original source
- claims are not exaggerated
- product names are correct
- feature availability is not overstated
- pricing, access, region limits, and release status are verified
- no copied wording from source articles
- no medical, legal, or financial advice
- no "AI will replace everyone" framing
- no claim that AI output is guaranteed accurate
- no auto-publishing

## Fact vs interpretation

Use this split in every brief:

### Confirmed

What the source directly says.

### Interpretation

What Clearforge thinks this may mean for practical users.

Never present interpretation as fact.

## Source handling

Prioritise:

- official company announcements
- official API or product documentation
- reputable technology journalism
- research papers or model/system cards where relevant
- primary documentation

Avoid relying on:

- screenshots without links
- social media rumours
- anonymous claims
- engagement-bait AI threads
- rewritten summaries that do not link to the original source

## Publishing statuses

Use one of these statuses:

- Draft
- Needs source check
- Needs claim check
- Ready for edit
- Approved for publishing
- Published

## Automation safety

Automation may collect sources, generate drafts, create assets, and prepare scheduled posts.

Automation must not publish live content unless a later human-approved publishing stage is explicitly built.

Default mode is review-first.
