# Clearforge Manual 7-Day Test Plan

## Goal

Test whether the Clearforge Daily AI Brief format can reliably turn AI news into useful articles and social content before building full automation.

## Day 1: Format test

- Pick 3 AI stories manually.
- Run the core prompt.
- Check whether the output follows the structure.
- Mark confusing sections.

Success measure:
The output is clear enough to edit into a publishable article.

## Day 2: Source discipline test

- Pick 3 stories from different sources.
- Check whether facts and interpretation stay separate.
- Add missing source links manually if needed.

Success measure:
No unsupported claims appear in the article.

## Day 3: Practical usefulness test

- Choose stories that affect creators or small businesses.
- Check whether the brief gives a real next action.

Success measure:
The "what to test next" section is useful and specific.

## Day 4: Repurposing test

- Generate social pack from the article.
- Check TikTok, Shorts, Facebook, Pinterest, LinkedIn, and card lines.
- Remove anything too salesy or hype-led.

Success measure:
At least 3 social formats are usable after light editing.

## Day 5: Speed test

- Time the manual workflow from source selection to final draft.
- Note bottlenecks.

Success measure:
Manual draft can be produced in a repeatable process.

## Day 6: Editorial risk test

- Look for overclaiming.
- Look for copied phrasing.
- Check whether medical, legal, or financial advice accidentally appears.
- Check whether model or tool availability is overstated.

Success measure:
The safety checklist catches issues before publishing.

## Day 7: System decision

Review:

- Was the article useful?
- Were sources easy to verify?
- Did the prompt produce too much waffle?
- Did the social formats save time?
- What should be automated first?

Decision options:

A. Keep manual for another week.
B. Create a semi-automated source collection script.
C. Build full daily workflow later.
D. Stop or reposition the product.

## Pass criteria

Move to automation only if:

- articles are useful after editing
- sources are reliable
- the claim-check list is working
- social outputs are reusable
- the workflow saves time rather than creating more admin
