# Clearforge YouTube Channel Notes

## Current position

The YouTube channel is fairly new and has little/no traction yet.

The API posting route reportedly works for posts/uploads.

## Implication

YouTube is not the first technical blocker.

The main YouTube risk is growth and packaging:

- weak opening hook
- low retention
- unclear viewer promise
- inconsistent topic lane
- no proven short-form format yet
- no clear reason for viewers to click through to the article

## Recommended Clearforge YouTube role

Use YouTube Shorts as a daily discovery channel, not the main product.

The main product remains the blog/article brief.

YouTube should point viewers toward the daily Clearforge lesson and practical test.

## First format to test

Daily 30 to 45 second Short:

1. Hook: "AI news is noisy. Here's what actually matters today."
2. One confirmed update.
3. One plain-English interpretation.
4. One small practical test.
5. Close: "Clearforge: human-led, AI-empowered."

## Posting strategy

Start with:

- one Short per daily brief
- same core idea as the blog article
- no overproduced visuals at first
- clear subtitles
- simple vertical format
- consistent title structure

## Measurement

Track:

- views after 24 hours
- average view duration
- swipe-away/drop-off if available
- comments/questions
- clicks to blog if available
- which hooks get repeated views

## Automation note

Because the API route already works, the next automation priority is not YouTube authentication.

Priority order:

1. Daily article draft system.
2. Social script generator.
3. Simple vertical video generator.
4. Human review gate.
5. API upload using existing working YouTube path.

## Safety rule

Do not publish a YouTube Short until the underlying article has passed source and claim checks.
